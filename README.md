# 慢病医防融合管理平台 MVP

这是一个零依赖的本地 MVP，用于快速验证慢病闭环管理模块。项目包含静态前端和 Node.js 本地后端，管理端数据与个人健康信息库统一落在本地 JSON 文件中。

## 打开方式

最简单方式：双击 `open-localhost.cmd`。它会启动服务窗口并打开：

```text
http://localhost:5173/
```

推荐启动本地服务：

```powershell
cd chronic-care-platform
npm.cmd run dev
```

然后打开：

```text
http://localhost:5173
```

如果你的 PowerShell 没有限制脚本执行，也可以使用 `npm run dev`。

如果浏览器提示 `localhost:5173` 无法打开，说明服务窗口没有运行。请重新双击 `open-localhost.cmd`，或者在命令行执行 `npm.cmd run dev` 后保持该窗口不要关闭。

备用方式：双击 `open-static.cmd`。它不需要启动服务，会直接打开 `index.html`，数据保存在浏览器本地。

如果只是看界面，也可以直接用浏览器打开 `index.html`，此时数据会保存在浏览器 `localStorage` 中。

## 已实现功能

- 工作台核心指标：建档人数、慢病人数、高危人数、待随访、逾期随访、控制率。
- 居民档案：新增、编辑、查询、按机构筛选、居民详情查看。
- 慢病登记：登记高血压、糖尿病、冠心病、脑卒中。
- 风险评估：根据收缩压、空腹血糖、BMI 自动分为低危、中危、高危。
- 随访管理：新增、记录、一键完成、逾期自动识别。
- 统计分析：病种统计、机构统计、闭环进展展示。
- 台账导出：导出居民慢病管理 CSV 台账。
- C 端居民端：从健康档案和电子病历时间线开始展示居民个人健康信息。
- 手机预览：通过 `mobile-preview.html` 在电脑中查看居民端手机视口效果。
- 个人健康信息库：归集健康档案、电子病历、检查检验、用药处方、过敏史、免疫接种、手术住院、授权共享。
- 居民自主管理：支持上传健康资料和新增授权记录，服务模式下保存到 `data/db.json` 的统一 `personalRecords` 数据模型。
- 居民端身份模型：支持个人账户和家庭成员切换，不再只是全量居民下拉选择。
- 授权共享闭环：授权记录支持有效期判断、过期提示和撤销授权。
- 医疗机构端：查看协同任务、授权健康档案、医保贯通提示。
- 医保端：查看慢病结算审核队列、医保支付金额、控费规则。


```text
http://电脑局域网IP:5173/citizen.html
```

## 数据说明

启动本地服务后，数据保存在：

```text
data/db.json
```

核心数据集合：

- `residents`：居民基础档案与健康指标。
- `accounts`：居民端个人账户和家庭成员绑定关系。
- `diseases`：慢病登记。
- `followups`：随访计划与记录。
- `personalRecords`：个人健康信息库，统一保存电子病历、检查检验、用药处方、过敏史、免疫接种、手术住院、授权共享、居民上传资料。
- `careOrders`：医疗机构端协同任务。
- `insuranceClaims`：医保端结算审核数据。

直接打开 `index.html` 时，数据保存在浏览器 `localStorage` 中。点击页面右上角“重置演示数据”可恢复内置样例数据。
