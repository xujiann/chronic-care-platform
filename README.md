# 健康城市四端协同系统 MVP

这是一个面向健康城市建设的开源 MVP，围绕卫生健康委端、医疗机构端、医保端、个人端四个系统，验证慢病管理、健康档案、电子病历、医疗资源监管、医保审核和固定取药等关键流程。

系统补充参考《“十四五”国家信息化规划》中关于普惠数字医疗、全民健康信息平台、医疗数据共享互认、互联网医疗监管、公共卫生应急数字化、基层智慧治理、数据安全和适老化服务的建设方向。

健康档案功能补充参考《健康档案基本架构与数据标准（试行）》，按“生命阶段、健康和疾病问题、卫生服务活动”三维架构组织居民档案，并把 32 类基础数据集映射到个人端和医疗机构端的查看视图。

项目包含静态前端和 Node.js 本地后端。本地服务优先使用 SQLite 主存储，并同步维护 `data/db.json` 作为 GitHub Pages 静态预览快照。

## 打开方式

最简单方式：双击 `open-localhost.cmd`。它会启动服务窗口并打开：

```text
http://localhost:5173/
```

推荐启动本地服务：

```powershell
cd chronic-care-platform
npm.cmd run dev
```

然后打开：

```text
http://localhost:5173
```

统一登录入口：

```text
http://localhost:5173/login.html
```

演示账号密码：

- 卫健委端：`whjw / 123456`
- 医疗机构端：`doctor / 123456`
- 医保端：`insurance / 123456`
- 个人端：`citizen / 123456`
- 县域医共体平台：`county / 123456`

统计导入任务接口：卫健委端登录后可调用 `POST /api/health-statistics/import-jobs` 登记 PDF/Excel 报表导入、统计直报系统接口或医疗机构接口对账任务，任务会进入 `healthStatisticsIngestion.jobs` 并写入安全审计。
使用 PowerShell 手工联调包含中文的 JSON 时，请用 UTF-8 字节提交请求体；浏览器 `fetch` 默认可正常提交 UTF-8。

如果你的 PowerShell 没有限制脚本执行，也可以使用 `npm run dev`。

如果浏览器提示 `localhost:5173` 无法打开，说明服务窗口没有运行。请重新双击 `open-localhost.cmd`，或者在命令行执行 `npm.cmd run dev` 后保持该窗口不要关闭。

备用方式：双击 `open-static.cmd`。它不需要启动服务，会直接打开 `index.html`，数据保存在浏览器本地。

如果只是看界面，也可以直接用浏览器打开 `index.html`，此时数据会保存在浏览器 `localStorage` 中。

## 已实现功能

- 卫生健康委端：监管总览、慢病医防整合、居民档案、慢病登记、随访管理、统计分析、协同监管、医疗资源监管、公共卫生应急、数据安全审计、国家规划对齐。
- 统一运营工作台：整体梳理健康城市系统结构、跨端待办、数据底座成熟度和继续开发优先级，作为后续逐项开发的总入口。
- 监管总览核心指标：建档人数、慢病人数、高危人数、待随访、逾期随访、控制率。
- 居民档案：新增、编辑、查询、按机构筛选、居民详情查看。
- 个人主索引：居民个人数据统一使用 `身份证号#手机号` 形成 `personIndex`，并同步到慢病、随访、电子病历、医保、取药等跨端数据。
- 电子凭证：个人端展示电子健康码、医保电子凭证、居民一卡通绑定状态，医保端进行医保凭证核验。
- 慢病登记：登记高血压、糖尿病、冠心病、脑卒中。
- 风险评估：根据收缩压、空腹血糖、BMI 自动分为低危、中危、高危。
- 随访管理：新增、记录、一键完成、逾期自动识别。
- 统计分析：病种统计、机构统计、闭环进展展示。
- 卫生健康统计：依据卫生健康统计工作管理要求，卫健委端新增医疗卫生资源和服务量统计，覆盖床位、医生数、护士数、门急诊量、住院量、出院量、床日，并按医疗机构接口与卫生健康统计直报系统双源对账。
- 统计公报可视化：将《2024 年我国卫生健康事业发展统计公报》关键数据转成卡片、领域概览、2023/2024 趋势对比和明细表。
- 台账导出：导出居民慢病管理 CSV 台账。
- C 端居民端：从健康档案和电子病历时间线开始展示居民个人健康信息。
- 手机预览：通过 `mobile-preview.html` 在电脑中查看居民端手机视口效果。
- 个人健康信息库：归集健康档案、电子病历、检查检验、用药处方、过敏史、免疫接种、手术住院、授权共享。
- 标准健康档案：个人端展示生命阶段、健康问题、卫生服务活动和 32 类标准数据集归集状态，居民可看到哪些信息已归集、哪些需要补齐。
- 档案病历贯通：个人端提供统一健康时间线，医疗机构端按授权查看健康指标、慢病、电子病历和检查检验摘要。
- 居民自主管理：支持上传健康资料和新增授权记录，服务模式下保存到 SQLite 主存储并同步到 `data/db.json` 的统一 `personalRecords` 数据模型。
- 居民端身份模型：支持个人账户和家庭成员切换，不再只是全量居民下拉选择。
- 授权共享闭环：授权记录支持有效期判断、过期提示和撤销授权。
- 医疗机构端：查看协同任务、授权健康档案、医保贯通提示。
- 医疗机构端标准档案视图：医生按授权查看患者标准健康档案归集度、适用数据集、慢病管理、门诊诊疗、检查检验、用药和待补齐项。
- 医疗机构端补充：慢病固定取药协同、授权与审计留痕。
- 医保端：查看慢病结算审核队列、医保支付金额、控费规则、医疗机构监管事项。
- 医保端补充：审核访问留痕，与卫健委端数据安全审计贯通。
- 县域医共体平台：依据《紧密型县域医共体信息化功能指引》建设 36 项功能清单，覆盖区域医技共享、便民惠民、医疗管理、公卫协同、基层综合管理。
- 医共体协同驾驶舱：展示县级总医院、乡镇卫生院、村卫生室、疾控/妇幼/急救中心的县乡村一体化组织网络。
- 医共体运营监管：展示基层首诊、县域内就诊、检验检查互认、慢病规范管理、家庭医生履约、医保协同审核等指标。
- 分级诊疗体系：依据国办发〔2026〕11号文件，实现基层首诊、双向转诊、预留号源床位、上转下转、家庭医生签约、医保差异化支付和长期处方闭环。
- 分级诊疗跨端贯通：卫健委端监管规则和转诊台账，医疗机构端承接转诊中心，医保端配置支付引导，个人端展示就医指引和转诊服务，县域医共体平台统筹建设。
- 协同监管：四端运行监测、风险预警、机构绩效、数据质量。
- 个人端慢病服务：每月固定取药计划、下次取药时间、药房、医保类型、申请方式、机构确认、医保审核和取药状态。
- 固定取药闭环：个人端申请、医疗机构确认、医保端审核、药房取药状态统一回流卫健委端。
- 国家规划对齐：把普惠数字医疗、医疗数据共享互认、互联网医疗监管、电子健康码/医保电子凭证方向、公共卫生应急、基层智慧治理、数据安全、适老化服务映射为系统建设任务。
- 公共卫生应急：多点触发预警、集中研判、资源调配、处置反馈。
- 适老服务：个人端展示家属代办取药、大字模式提醒、线下帮办预约等服务。
- 个人端访问透明：居民可查看自己的健康信息访问记录，了解访问方、用途、范围和结果。
- 数据安全审计：展示授权状态、访问留痕、拒绝访问、脱敏策略和合规规则。
- 统一登录与接口权限：本地服务下支持后端登录、Bearer 会话、写接口角色校验、退出注销、安全事件日志；静态预览自动降级为前端演示会话。
- 系统流程结构图：见 `docs/system-flow.md`，包含整体架构、慢病闭环、档案病历贯通、登录权限流程。

## 数据说明

启动本地服务后，优先使用 SQLite：

```text
data/health-city.sqlite
```

同时维护静态预览快照：

```text
data/db.json
```

核心数据集合：

- `residents`：居民基础档案、健康指标和 `personIndex`。
- `accounts`：居民端个人账户和家庭成员绑定关系。
- `diseases`：慢病登记，关联 `residentId` 与 `personIndex`。
- `followups`：随访计划与记录，关联 `residentId` 与 `personIndex`。
- `personalRecords`：个人健康信息库，统一保存电子病历、检查检验、用药处方、过敏史、免疫接种、手术住院、授权共享、居民上传资料，并关联 `personIndex`。
- `healthArchiveStandard`：健康档案基本架构与数据标准目录，包含三维架构、内容分组和 32 类基础数据集。
- `medicalResources`：卫生健康委端医疗资源数据。
- `healthStatistics`：卫生健康统计数据，包含医疗机构接口、卫生健康统计直报系统、资源对账、诊疗量、住院量和质量规则。
- `healthBulletin2024`：2024 年卫生健康事业发展统计公报可视化数据，包含关键指标、领域概览、趋势对比和明细指标。
- `dalianHealthStatistics2025`：2025 年大连市卫生健康统计提要结构化摘要，包含大连关键指标、与国家公报对比、报表导入和统计直报系统接入路径。
- `healthStatisticsIngestion`：卫生健康统计数据接入流程，覆盖报表采集、解析、指标映射、质控、入库、发布和审计留痕。
- `storageMeta`：本地 SQLite 主存储和 GitHub Pages JSON 快照的存储状态说明。
- `careOrders`：医疗机构端协同任务。
- `medicationPickups`：个人端每月固定取药计划。
- `institutionSupervisions`：医保端医疗机构监管事项。
- `insuranceClaims`：医保端结算审核数据。
- `policyAlignment`：国家信息化规划要求与本系统能力落点映射。
- `emergencySignals`：公共卫生应急多点触发预警。
- `seniorServices`：适老化、家属代办、线下帮办等居民服务。
- `dataAccessLogs`：跨端访问个人健康信息的审计日志。
- `securityEvents`：登录、退出、接口拒绝、敏感写操作等安全事件日志。
- `digitalCredentials`：电子健康码、医保电子凭证、居民一卡通等数字凭证。
- `authUsers`：演示登录角色目录，真实部署时应改为后端认证和权限系统。
- `countyConsortium`：县域医共体平台数据模型，包含组织网络、36 项功能清单、重点任务、业务闭环、运营指标和治理要求。
- `referralSystem`：分级诊疗数据模型，包含机构功能定位、转诊规则、转诊台账、预留号源床位、医保支付引导、家庭医生签约和居民宣传教育。
- `platformRoadmap`：全系统继续开发优先级，包含 P0/P1/P2、建设原因、涉及范围、当前状态和下一步动作。
