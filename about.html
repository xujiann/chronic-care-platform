可直接运行的角色端

同一套演示账号和权限过滤，支持本地 API 与静态预览降级
统一运营工作台 查看系统成熟度、发布证据门禁、跨模块待办、服务验收摘要、模板 README、全流程审计与发布产物。 卫健委管理端 运行监管总览、慢病医防整合、居民档案、慢病登记、随访管理、资源监管、绩效、风险预警、数据质量和系统运行监测。 医疗机构端 处理诊疗协同、签约居民、随访、长期处方、多点执业、固定取药、死亡医学证明和机构侧任务。 医保管理与经办端 演示医保待遇协同、结算经办、凭证核验、固定取药审核、基金监管和区县医保协同。 县域医共体平台 展示县域资源共享、基层慢病、公共卫生协同、统一绩效、人财物管理和医共体服务清单。 居民端与手机预览 居民可查看个人健康信息库、家庭成员、授权共享、电子病历、检验检查、用药和固定取药信息。

登录与权限

演示密码统一为 123456
统一登录通过 /api/auth/login 签发后端会话令牌；静态预览时自动降级为本地演示会话。
角色路由auth.js 按角色过滤导航和页面访问，居民端不会暴露管理端入口。
接口授权前端通过 authFetch 携带令牌访问受保护 API，服务端记录登录和登出审计。

当前可运行的能力

面向演示、验收、审计和继续开发的实际功能面
统一认证与状态接口 可调用 /api/auth/login、/api/auth/me、/api/state，并按用户组织范围返回数据。
统一任务与开放动作 可调用 /api/tasks 汇总慢病、医共体、上线验收、模板和服务补齐任务，工作台支持一键打开相关证据。
基层慢病管理 已实现慢病服务角色、能力条件、服务路径、多病共管、中医药服务、自我管理、用药支持和质量指标看板。
县域医共体协同 已覆盖医共体成员机构、资源共享、基层服务、公共卫生协同、绩效管理和县域服务验收闭环。
服务验收摘要 可调用 /api/service-acceptance-summary，输出各服务域、未关闭动作、证据路径和工作台跳转入口。
模板 README 可调用 /api/site-template-readmes 查看已按当前功能重写的模板说明，并纳入站点就绪包。
发布与审计门禁 可运行 npm run check、npm test、npm run test:e2e、npm run deploy:check、npm run release:report。
集成与外部接口准备 已建成接口映射、集成就绪、生产库就绪和监控就绪报告；真实上线需替换统一身份、HIS/EMR/LIS/PACS、医保和短信等外部系统。
