# 全民健康信息平台开发任务清单

本清单依据《大连市全民健康信息平台建设及能力提升项目提级论证申报材料（整合修订版）》拆解，目标是在现有 `chronic-care-platform` 项目基础上继续开发，不另起一套割裂系统。

## 一、已纳入本次开发的入口

- 新增 `platform.html`：市级全民健康信息平台建设驾驶舱。
- 新增 `platform.js`：申报功能覆盖矩阵、存量系统整合台账、接口衔接清单、开发批次。
- 已将 `platformCapabilities`、`platformIntegrations`、`platformInterfaces`、`platformDeliveryBatches` 纳入后端 `seedState` 和 `normalizeState`，旧版 `data/db.json` 会自动补齐这些建设管理数据。
- 已在 `platform.html` 增加建设项维护弹窗，可维护建设域、存量整合、接口衔接、开发批次的状态、责任方和下一步动作，并通过 `/api/state` 回写。
- 已新增 `platformChangeLogs`，平台建设项维护会自动沉淀最近维护记录，用于周报、验收取证和责任追踪。
- 已新增“周报素材”区块，按状态、责任方、建设类别和维护时间生成推进摘要，并支持筛选后一键导出 Markdown。
- 已新增 `platformEvidence` 验收证据库，按申报材料、互联互通测评、安全合规、接口联调和上线验收归档证据项。
- 已新增验收证据登记弹窗，支持维护文件名、截图/链接、测试记录、责任人和整改闭环状态，并自动写入最近维护记录和周报素材。
- 复用 `shared.js` 的 `loadPlatformState`，继续读取 `/api/state` 或 `data/db.json`，与慢病、医共体、居民端、机构端、医保端共享同一数据底座。

## 二、主要功能覆盖

- 城市级医疗健康大数据平台：统一平台底座、区域医疗健康大数据中心、全域互联互通、数据资产管理、信创及国产密码改造。
- 助医应用：分级诊疗、远程会诊、双向转诊、远程影像、远程心电、委托检验、远程教育、临床治疗辅助、居民健康数字身份。
- 惠民应用：健康大连互联网应用统一入口、互联网+药事服务、居民健康画像、授权共享、固定取药提醒。
- 辅政应用：数智健康大脑、卫生健康统计数据质控和共享、医疗机构信用评价。
- 科研创新：专病库、多模态医疗数据集、科研数据授权、脱敏和发布。
- 区级实施：区属医院、基层机构、妇幼机构、体检机构接入，以及市级应用下沉。
- 互联互通测评：共享文档、术语标准、主索引、交易服务、文审材料、模拟演练、现场查验证据。
- 安全信创：等保三级、密评、国产密码、统一认证、访问审计、信创适配。

## 三、与原有项目衔接

- 慢病模块继续承担筛查、建档、风险分级、随访、宣教、固定取药和专病库基础能力。
- 医共体模块继续承担县乡村一体化、医技共享、互认、基层 AI 辅助诊断和协同工单。
- 机构端继续承担授权健康档案、转诊接诊、出生/死亡医学证明、医生多点执业等业务。
- 医保端继续承担医保审核、凭证核验、固定取药审核和基金监管。
- 居民端继续承担个人健康信息库、家庭成员、授权共享、用药和取药服务。
- 运营工作台继续承担跨端待办、流程审计、数据成熟度和开发优先级管理。

## 四、下一步开发建议

1. 建立统一应用目录和存量模块登记表，给每个模块标注来源系统、接口方式、责任处室、复用方式和验收证据。
2. 新增医疗机构信用评价数据模型和页面，将辅政应用补齐为可操作闭环。
3. 新增互联互通测评证据库，按四甲、五乙要求挂接标准文档、接口、截图、测试记录和整改状态。
4. 把安全信创清单细化为等保、密评、国密改造、信创适配四类验收台账。
5. 增加统一应用目录，按来源系统、接口方式、责任处室、复用方式、上线批次和验收证据追踪存量模块。
