# 全民健康信息平台下一步开发规划

更新时间：2026-06-23

当前系统已经形成多端演示级闭环：慢病医防融合、居民端、医疗机构端、医保经办、县域医共体、区域数据共享、质量安全、医院运行调度、药耗监管、科研数据集、统一运营工作台和综合驾驶舱均有页面、数据集合、API 或发布证据。下一阶段不再以“新增展示页面”为主，而应围绕互联网预览、真实试运行和现场验收补齐生产底座。

## 1. 当前发布目标

| 目标 | 当前状态 | 下一步动作 |
|---|---|---|
| GitHub PR 预览 | 已推送 `codex/p0-test-ci-foundation` 分支，PR #2 持续汇总完整系统能力 | 等 CI 通过后保持 PR 作为完整系统审查入口 |
| 互联网静态预览 | GitHub Pages 发布源为 `main` 分支 | 将通过审查后的完整系统同步到 `main`，再验证 Pages 站点 |
| 系统流程材料 | 已生成 `docs/system-flow.md` 与 PDF 版流程图 | 后续把流程图作为汇报、评审和现场实施沟通材料 |
| 发布证据 | 本地 `check`、`test`、`deploy:check`、`release:report`、`release:manifest` 已通过 | CI artifact 与现场签字材料继续沉淀到 release evidence |

## 2. P0：互联网预览与试运行底座

| 任务 | 内容 | 验收标准 |
|---|---|---|
| 公开预览校验 | 验证 GitHub Pages 首页、登录页、各角色入口、综合驾驶舱、流程图 PDF 链接 | 桌面和移动端可访问，无 404，无明显版式错位 |
| 静态/后端边界说明 | 在预览站明确哪些能力是静态快照，哪些必须依赖 Node API | 用户不会把 GitHub Pages 误认为生产系统 |
| 生产环境矩阵 | 继续细化 demo、staging、production 三套环境参数 | `env:check:production` 能指出所有现场待补项 |
| 发布包清单 | 扩展 release manifest，索引流程图 PDF、综合驾驶舱证据、接口联调模板 | 发布材料可被项目办和实施组直接复用 |

## 3. P0：真实身份、权限和审计

| 任务 | 内容 | 验收标准 |
|---|---|---|
| 政务身份源适配 | 对接 OIDC/统一认证、机构目录、医生身份、居民实名关系 | 演示账号退出生产路径，角色与组织来自真实身份源 |
| 字段级授权 | 居民档案、病历、检查、医保、证照按角色和授权范围脱敏 | 跨角色、跨机构、越权访问自动拒绝并留痕 |
| 授权闭环深化 | 授权、撤销、过期、访问历史复核与居民端展示一致 | 授权撤销后历史访问可复核，新增访问被拦截 |
| 审计保全 | 接入 SIEM 或审计导出目录，固化哈希链保全机制 | `release:report` 不再出现审计保全目标警告 |

## 4. P1：外部接口联调

| 任务 | 内容 | 验收标准 |
|---|---|---|
| 医疗机构接口 | HIS、EMR、LIS、PACS、心电、体检系统字段映射和样例报文 | 至少贯通一家机构的患者、病历、检验、影像和转诊链路 |
| 医保接口 | 门慢门特、固定取药、基金监管、结算审核规则联调 | 医保审核与取药状态能回写平台并可对账 |
| 公卫和统计 | 家庭医生签约、公卫随访、疾控、卫生统计直报 | 慢病管理、随访和统计指标来自真实业务数据 |
| 电子证照 | 出生、死亡、数字健康凭证与电子证照平台联通 | 证照申请、审核、签发、查询形成闭环 |

## 5. P1：业务闭环深化

| 任务 | 内容 | 验收标准 |
|---|---|---|
| 慢病试点闭环 | 选择高血压、糖尿病作为首批专病试点 | 筛查、登记、分层、随访、用药、居民反馈、质控全链路可追踪 |
| 县域医共体试点 | 双向转诊、检查互认、共享中心、基层履约和绩效质量联动 | 县乡村协同任务有责任人、状态、时限、审计 |
| 质量安全监管 | 危急值、不良事件、路径偏离、整改反馈、专家复核 | 问题派单到复核闭环有证据、有时限 |
| 医院运行调度 | 床位、人力、设备、急诊、药耗、统计直报对账 | 运行指标可实时刷新并支撑调度动作 |

## 6. P2：治理、科研和用户体验

| 任务 | 内容 | 验收标准 |
|---|---|---|
| 机构信用与绩效 | 信用扣分、申诉、公示、医共体绩效、药耗和基层履约 | 评分规则版本化，结果可解释、可申诉、可复核 |
| 科研数据集 | 伦理审批、脱敏、沙箱访问、使用审计、成果回流 | 数据集全生命周期有审批和审计记录 |
| 移动与适老化 | 大字模式、读屏语义、家属代办、线下帮办、弱网模式 | 老年用户关键任务通过现场可用性测试 |
| 运营可视化 | 综合驾驶舱聚合八类重点应用边界、证据和风险 | 领导查看总览，业务人员可下钻到源应用 |

## 7. 推荐实施顺序

1. 先完成互联网预览验证和 Pages 发布，锁定当前演示边界。
2. 再补生产切换证据：身份、接口、审计、监控、灾备和现场签字模板。
3. 选择一家医疗机构和一个区县做联调试点，先贯通慢病和转诊链路。
4. 扩展医保、公卫、证照和统计接口，形成区域闭环。
5. 最后深化科研、信用绩效、适老化和运营驾驶舱。
