# 互联网护理上线与下一步开发计划

## 上线标准

### 角色闭环

- 居民手机端：只能提交本人或家庭授权居民的预约，预约来源标记为 `internet-nursing-mobile`，不能伪造护士、评估结论、知情同意或完成状态。
- 医院端：完成首诊评估、知情同意、风险复核、合格护士派单和质控回访。
- 护士手机端：只能处理本人工作站或本机构未派单任务，按接单、开始服务、完成记录顺序推进。
- 卫健管理端：查看机构、护士、订单、风险队列、位置轨迹和 readiness 证据。

### 上线门禁

- 服务目录：预约项目必须在试点政策目录和机构公开服务目录内。
- 服务对象：预约对象必须属于试点服务对象范围。
- 风险控制：高风险订单必须进入风险提示，未完成首诊评估和知情同意不得接单。
- 证据留痕：预约、派单、接单、开始服务、完成记录、质控回访必须写入审计轨迹和任务消息。
- 移动体验：390px 手机视口下展示预约和接单卡片，桌面表格不作为居民和护士主入口。
- 发布证据：`internet-nursing:readiness`、静态测试、API 聚焦测试、`deploy:check` 必须通过。

## 已开发功能清单

### 个人端手机预约

- 已提供居民手机端预约入口，支持选择试点机构、护理服务项目、服务对象、期望时间和上门地址。
- 已接入服务目录校验，预约项目必须属于政策目录和机构公开目录。
- 已按本人及家庭授权范围裁剪居民数据，防止越权代约。
- 已将预约来源标记为 `internet-nursing-mobile`，便于后续统计手机端转化和运行质量。

### 医院端管理

- 已提供医院端订单总览、风险提示、首诊评估、知情同意、派单和质控回访动作。
- 已按试点机构、服务目录、护士资质和订单状态进行管理闭环。
- 已保留任务消息和审计轨迹，支持监管端追溯预约、派单、回访等关键节点。

### 护士端手机接单

- 已提供护士工作站演示账号 `nurse / 123456`，登录后锁定护士身份和可处理订单。
- 已支持手机端接单、开始服务、完成护理记录的状态推进。
- 已限制护士只能操作本人或本机构可处理订单，避免跨机构、跨护士越权。
- 已在移动卡片展示风险等级、评估状态、知情同意、护理记录附件数、消息回执和下一步操作。
- 已补齐结构化护理记录，完成服务时可写入护理措施、生命体征、耗材使用、居民状态、随访建议、照片/签名附件和异常事件状态。

### 监管与上线证据

- 已提供卫健管理视角的机构、护士、订单、风险队列、轨迹订单和证据完整性统计。
- 已提供 `GET /api/internet-nursing/dashboard`、`POST /api/internet-nursing/orders`、`POST /api/internet-nursing/orders/:id/actions` 三类核心接口。
- 已纳入 `internet-nursing:readiness`、静态测试、发布检查和 CI 验证。
- 已补齐中文可见文本校验，防止页面和发布证据出现乱码或占位文本。

## 三阶段开发完成情况

### 阶段一：上线联调（已完成）

- 已接入互联网护理消息网关投递证据，预约受理、派单、护士接单、服务开始、服务完成和回访结果均生成站内消息、短信、院内消息三类触达记录；当前短信和院内消息为 `simulation` 模式，具备生产网关替换契约。
- 已将护士定位轨迹升级为 `locationTracePoints` 轨迹点列表，护士接单、开始服务、完成记录时均可追加手机端定位点，用于服务开始和结束位置核验。
- 已增加居民端知情同意电子签名附件 `consentAttachment`，医院端评估后可核验签署时间、签署人、版本、附件名和哈希摘要。

### 阶段二：运营管理（已完成）

- 已增加医院端派单推荐，按护士资质、服务项目、所属机构、服务区域、日容量、已派单量和风险等级生成候选护士。
- 已增加费用规则、医保/自费结算预估、支付状态和居民端费用质量展示，支撑服务价格、医保预核和自费估算。
- 已增加满意度、投诉状态、质控抽查和不良事件字段；质控回访动作会补齐满意度和抽查记录，监管月报据此计算机构质量评分。

### 阶段三：监管扩展（已完成）

- 已增加试点机构准入审核、服务目录变更审批和护士资质到期提醒，并在监管扩展面板集中展示。
- 已建立卫健管理端月报，覆盖服务量、完成量、风险处置、回访闭环、投诉率、不良事件、轨迹完整率和机构质量评分。
- 已输出监管接口契约 `internet-nursing-regulatory-contract-v1`，覆盖 `/api/internet-nursing/dashboard`、`/api/internet-nursing/orders`、`/api/internet-nursing/orders/:id/actions`，供院内护理管理系统、电子病历、医保结算和监管平台对接。

## 本轮上线增强

- 已把三个阶段的完成状态纳入 `internet-nursing:readiness`：`nursing:phaseOneEvidence`、`nursing:notificationGateway`、`nursing:phaseTwoOperations`、`nursing:phaseThreeRegulation` 均为发布必过门禁。
- 已在页面增加“派单推荐”“费用与质量”“监管月报”“准入与接口契约”四个可运行面板，不再停留在文档规划。
- 已在服务端 dashboard 返回 `dispatchRecommendations`、`regulatoryMonthlyReport`、`regulatoryAlerts`、`regulatoryContract`，并保留静态预览兜底。
- 已增加 `nursing:serviceRecordClosure` readiness 门禁，覆盖结构化护理记录、服务附件、异常上报、消息回执和护士手机端展示。
- 已增加生产集成、真实支付、设备核验和监管报送四项闭环对象：`productionIntegration`、`paymentReadiness`、`deviceVerification`、`regulatorySubmission`。
- 已增加 `nursing:productionIntegration`、`nursing:paymentIntegration`、`nursing:deviceVerification`、`nursing:regulatorySubmission` readiness 门禁，覆盖生产消息网关、电子签名附件存储、医保电子凭证、移动支付、退费、发票、对账、定位设备、记录仪、一键报警、字段映射、现场签字和压测证据。

## 生产化闭环完成情况

- 生产集成：已建立 `internet-nursing-production-integration-v1`，覆盖短信、院内消息、站内消息、电子签名附件存储、院内护理管理系统、电子病历和卫健监管平台连接器；页面提供“生产集成联调”面板。
- 真实支付：已建立 `internet-nursing-payment-v1`，覆盖医保电子凭证预核、移动自费支付、退费、电子发票和 T+1 日终对账；页面提供“支付对账与票据”面板。
- 设备核验：已建立 `internet-nursing-device-verification-v1`，覆盖手机 GPS、护士定位设备、服务记录仪、一键报警、照片附件和异常升级；页面提供“设备核验与附件”面板。
- 监管报送：已建立 `internet-nursing-regulatory-submission-v1`，覆盖机构、护士、订单、风险、轨迹、结算、质控和不良事件字段映射，并纳入月报、高风险实时、现场签字和压测证据；页面提供“监管报送签字压测”面板。
