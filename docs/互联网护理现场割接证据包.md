# 互联网护理现场割接证据包

本文件用于承接 `nursing:siteCutoverPack` readiness 门禁。它不替代真实现场签字，而是把互联网护理模块正式上线前需要归档的联调、压测、审计和签字证据拆成可执行清单。

## 割接轨道

| 轨道 | 责任方 | 需归档证据 |
| --- | --- | --- |
| `nursing-cutover-message-signature` | 医院护理信息化与平台运维 | 短信、院内消息、站内兜底、电子签名附件存储联调签字 |
| `nursing-cutover-hospital-connectors` | 医疗机构接口联调组 | 院内护理管理系统、电子病历、卫健监管平台连接器联调签字 |
| `nursing-cutover-payment-reconciliation` | 医保经办、财务与平台支付组 | 医保电子凭证、自费支付、退费、发票、T+1 对账验收签字 |
| `nursing-cutover-device-drill` | 护理运营与设备管理组 | 手机 GPS、护士定位设备、服务记录仪、一键报警、照片附件现场实测签字 |
| `nursing-cutover-regulatory-pressure-test` | 卫健监管与平台运维 | 月报、高风险实时上报、监管字段映射和压测记录签字 |

## 关联发布证据

- 现场签字模板：`release/templates/production-signoff/README.md`
- 审计留存报告：`release/audit-retention-report.md`
- 生产割接清单：`release/production-cutover-checklist.md`
- 模块 readiness：`release/internet-nursing-readiness-report.md`

## 上线注意

当前系统侧已经完成生产消息网关、电子签名附件存储、支付对账、设备核验和监管报送的契约化闭环。正式生产上线仍需配置 `AUDIT_EXPORT_PATH` 或 `SIEM_ENDPOINT`，并由试点医院、卫健监管、医保经办和平台运维完成现场签字、密钥托管、压测报告和灾备演练归档。
