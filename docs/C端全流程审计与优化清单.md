# C 端全流程审计与优化清单

更新日期：2026-06-18

本清单用于审计居民端从建档到使用服务的完整链路，并标明当前系统已实现、演示可用和待生产化的内容。

## 1. 全流程状态

| 流程 | 当前状态 | 说明 |
|---|---|---|
| 登录进入居民端 | 已实现 | 使用 `citizen / 123456` 演示账号 |
| 读取居民档案 | 已实现 | 从平台状态数据中读取居民、慢病、随访信息 |
| 查看健康档案 | 已实现 | 展示基本信息、指标、慢病标签和风险等级 |
| 查看电子病历 | 已实现 | 展示门诊/住院时间线、诊疗摘要、诊疗详情、医嘱和来源机构 |
| 查看检查检验 | 已实现 | 展示项目、结果、来源和时间 |
| 查看用药处方 | 已实现 | 展示长期用药、固定取药和处方摘要 |
| 查看随访计划 | 已实现 | 展示待随访、完成和逾期状态 |
| 授权共享 | 演示可用 | 可记录授权对象和范围，尚未成为强制权限模型 |
| 手机预览 | 已实现 | `mobile-preview.html` 可模拟移动端，居民端已补 PWA 安装入口和离线壳 |
| 多系统真实接入 | 未完成 | 需要对接 EMR、LIS、PACS、医保、公卫等系统 |

## 2. 当前主要风险

- 居民端身份仍是演示账号，缺少实名校验、手机号绑定和家庭成员授权。
- 授权共享缺少后端强制拦截、撤销后复核和访问日志联动。
- 健康信息来自演示数据，尚未建立真实接口映射、数据质量校验和来源可信度规则。
- 电子病历、检查检验、用药、影像和附件已形成演示级详情与归集入口；真实 PACS/文档存储授权、原文预览和下载仍需现场系统接入。
- 静态模式使用浏览器本地数据，服务模式使用 `data/db.json`，正式部署前应统一到数据库。

## 3. 优化优先级

| 优先级 | 优化项 | 目标 |
|---|---|---|
| P0 | 居民身份模型 | 支持本人、家庭成员、监护人和签约医生关系 |
| P0 | 授权闭环 | 授权、撤销、过期、访问日志和审计联动 |
| P1 | 电子病历详情 | 已补齐诊疗详情、医嘱摘要、检查/用药标签和来源机构 |
| P1 | 健康指标趋势 | 已展示血压、血糖、BMI 等核心指标趋势 |
| P1 | 数据来源标识 | 已标明医院、基层、公卫、医保或个人上传 |
| P2 | 附件和影像 | 已支持影像资料、附件资料、文件名、类型和调阅方式演示归集 |
| P2 | PWA 或小程序 | 已补 PWA manifest、图标、service worker 和离线回退壳 |

## 4. 验收建议

居民端每次迭代至少检查：

- 登录后是否能进入居民端。
- 居民档案、慢病、随访、病历、检查、用药是否正常展示。
- 手机预览是否无明显遮挡、溢出和按钮不可点问题。
- 授权记录是否能被创建、显示并进入审计视角。
- 数据缺失时是否有明确空状态，而不是页面报错。
