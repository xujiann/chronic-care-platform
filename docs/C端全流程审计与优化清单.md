## 2. 当前主要风险

- 居民端已支持手机号验证码演示登录，但仍缺少实名核验、真实短信网关、监护人关系核验和生产级风控。
- 授权共享已可在居民端创建和展示，仍缺少后端强制拦截、撤销后复核和访问日志联动。
- 健康信息来自演示数据，尚未建立真实接口映射、数据质量校验和来源可信度规则。
- 电子病历、检查检验、用药、影像和附件已形成演示级详情与归集入口；真实 PACS/文档存储授权、原文预览和下载仍需现场系统接入。
- 静态模式使用浏览器本地数据，服务模式使用 `data/db.json`，正式部署前应统一到数据库。

## 4. 验收建议

居民端每次迭代至少检查：

- 登录后是否能进入居民端。
- 手机号验证码登录是否能进入居民端，并且登录入口在手机端可触控。
- 居民档案、慢病、随访、病历、检查、用药是否正常展示。
- 居民端功能审计面板是否覆盖健康档案、电子病历、护理、陪诊、挂号，并能区分已实现和待开发。
- 手机端底部服务导航是否可随时切换健康档案、电子病历、护理、陪诊、挂号，并跳转到对应服务内容。
- `mobile-preview.html` 是否能在 390px 手机框中加载居民端，并通过预览控制切换健康档案、电子病历、护理、陪诊和挂号。
- 手机预览是否无明显遮挡、溢出和按钮不可点问题。
- 授权记录是否能被创建、显示并进入审计视角。
- 数据缺失时是否有明确空状态，而不是页面报错。
